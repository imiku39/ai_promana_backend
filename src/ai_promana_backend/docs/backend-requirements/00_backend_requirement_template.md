## 14. 错误码约定

| 错误码 | 含义 | 前端处理 |
| --- | --- | --- |
| 0 | 成功 | 正常渲染 |
| 400 | 参数错误 | 展示字段错误或 Toast |
| 401 | 未登录 | 跳转登录 |
| 403 | 无权限 | 展示无权限提示 |
| 404 | 数据不存在 | 展示空状态 |
| 500 | 服务异常 | 展示失败提示 |

## 15. 验收标准

- 页面首屏数据可通过接口完整渲染。
- 页面中的新增、编辑、删除、导入、导出等操作均有明确接口支撑。
- 接口字段与前端展示字段一致，不需要前端额外猜测或硬编码。
- 权限不足、空数据、接口失败、参数错误均有明确返回结构。
- 后端返回的枚举值、状态值、颜色语义与前端展示保持一致。
